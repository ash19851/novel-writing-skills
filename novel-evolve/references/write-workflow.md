# Novel Write — 写作执行引擎参考

## 一次写作循环的完整流程

```
① 加载状态 → ② 写作执行 → ③ 即时回填 → ④ 反哺分析 → ⑤ 方向讨论
```

## 字数目标
- 初始默认值：2500字/章
- 每章写完后必须核验，不足必须扩充

## 回填操作标准

每次回填时，在画布对应条目的末尾追加记录：

```markdown
- [第N章] <发现的细节> → <对条目的影响>
```

### 回填示例

**角色回填：**
```markdown
- [第3章] 主角在紧张时用手指敲桌面 → 追加习惯：敲击节奏对应思考深度
- [第5章] 主角被人质疑时先沉默再回答 → 追加特征：对抗性对话中的延迟反应
```

**地点回填：**
```markdown
- [第2章] 港口区有一棵歪脖子榕树，被当地人当路标 → 追加地标
- [第4章] 台风过后街上的积水需要半天才能退 → 追加城市细节：排水系统老旧
```

## 大纲修订原则

1. **正文优先**：已写出来的正文是不可否认的现实，大纲根据正文调整
2. **只修当前**：只关注当前和下一个事件，不提前修订更后面的内容
3. **保守修改**：大纲仍然合理就不为了改而改
