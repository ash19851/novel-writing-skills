# 探索式协作写作引擎 Novel Evolve

## 核心哲学

这套技能和传统的"先画蓝图→锁大纲→一路写到完"完全不同。

**你不需要在动笔前知道第300章会发生什么。你只需要知道第1章发生了什么，然后让故事自己告诉你下一步。**

## 适用场景

- "我们来一起构思一个故事"
- "我想试试探索式写作"
- "不要锁大纲，一边写一边发展"
- 任何对"先聊再写、边写边发现"创作方式的兴趣

## 初始化

首次使用时，在项目根目录下创建画布目录结构：

```
mkdir -p .canvas/{world,characters,plot,meta}
touch .canvas/meta/session-status.md
touch .canvas/meta/direction.md
```

## 工作流程

### Phase 0：状态感知
每次对话开始时，系统自动判断当前所在阶段，告知用户已完成的内容和画布条目数量。

### Phase 1：选题海聊（无压探索）
从一个模糊的想法出发，通过对话逐步收敛到可操作的大方向。不要立刻输出成型的选题方案，用追问引导用户自己表达。

### Phase 2：逐块搭建（6块积木）
顺序固定：**世界观 → 主角 → 宏观背景 → 入口 → 配角 → 性格**

每块聊透后再进入下一块，每块完成后立即写入画布。

### Phase 3：触发事件（故事启动）
通过对话触发一个具体的"故事启动事件"——不是一个章节大纲，而是一个场景、一个画面、一个冲突。

### Phase 4：章节大纲 + 正文书写（演化循环）
核心循环：**大纲生成 → 正文书写 → 即时回填 → 大纲修订**

发现的新细节（角色性格、环境特征）即时追加到画布中，让故事持续进化。

### Phase 5：卷收尾判断
在合适的时机判断是否结束当前卷，生成正式分卷大纲文档。

## 画布文档

所有设定存储在 `.canvas/` 目录下：

```
.canvas/
├── world/           # 世界观、地点
├── characters/      # 角色（持续进化）
├── plot/            # 事件链、剧情单元
└── meta/            # 方向、状态
```

详见 `references/canvas-format.md`

## 关键原则

1. **让用户先出牌** — 开放式问题优先，用户卡住时再提供选择
2. **修正不是反复** — 用户的中途修正说明讨论在深入
3. **具体比抽象有用** — 追问感官细节（光线、声音、气味）
4. **回填不能跳过** — 每章写完后必须先回填再继续
5. **正文优先** — 大纲根据正文调整，而非强行符合大纲
