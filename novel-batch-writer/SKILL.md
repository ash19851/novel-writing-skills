# 迭代批量写作器 Novel Batch Writer

## 适用场景

- 全本大纲（分卷+剧情单元）已完成
- 核心设定已定稿
- 用户说："批量写"、"5章一批"、"继续写"、"反复迭代直至全部完成"

## 前置文件清单

- `1-选题.txt`
- `2-核心设定.txt`
- `3-标签简介.txt`
- `4-分卷大纲.txt`
- `5-第X卷-剧情单元.txt`

## Part A：分章大纲批量生成

### Phase A1：确定批次位置
问"为第几章开始？"，计算批次范围 N~N+4。

### Phase A2：加载上下文
加载选题、设定、分卷大纲、剧情单元、上一批结尾钩子。

### Phase A3：生成5章大纲（推荐execute_code一次性生成）
每章模板：标题、核心目标、单元出处、场景、人物、关键情节(3-4节点)、爽点、结尾钩子。

约束：5章上限、剧情单元锁定、不新增角色、钩子链连续。

### Phase A4：强制批量核验（不可跳过）
核验维度：单元覆盖率、角色合规、爽点、钩子链、末章钩子匹配。
覆盖率<85或任一章有遗漏→退回修改。

### Phase A5：保存并确认
通过后保存到 `5-第{N}-{N+4}章-分章大纲.txt`。

## Part B：正文批量写作

### Phase B1：确定正文批次位置
确认对应分章大纲文件存在。

### Phase B2：写正文（逐章，强制2500+字）
每章写完后立即核验字数（不含标题和【本章完】标记）。
不足2500必须返回扩充，不得跳过。

### Phase B3：批量核验
5章全部写完后执行6维核验。

### Phase B4：修正或继续
匹配度≥80继续，<80用novel-editor修复。

### Phase B5：卷完成
卷级检查，确认所有剧情目标达成。

## 输出文件
- `5-第{N}-{N+4}章-分章大纲.txt`
- `8-正文-第{N}章.txt`
- `7-批量核验-第{N}-{N+4}章.txt`

## Batch Continuation Mode（批量延续模式）

当用户明确"继续直到结束"时，跳过确认直接进入下一批。
核验仍须执行（可精简为摘要格式），不可跳过。

## 常见陷阱
1. 多章合并写在一个文件中再拆分（每章字数不足）
2. 字数核验延迟（写一章核验一章，不累积债务）
3. 大纲文件范围重叠（每批独立范围文件）
4. Batch Mode中跳过核验（不可跳过）
